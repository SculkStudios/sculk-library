## What is implemented today

- Command DSL and Java command builder.
- GUI menu DSL, sessions, pagination, confirm menus, and platform cleanup.
- Item builder APIs with PDC helpers, skulls, and config descriptors.
- MiniMessage, Adventure messaging helpers, and reusable message templates.
- Typed YAML config loading, comments, validation annotations, strict mode, and reload results.
- JDBC repositories, async facades, storage config, ORM mapping, player profile helpers, and cache helpers.
- Registry-backed lookup helpers, require lookups, validation reports, and curated aliases through `sculk-series`.
- Particle, sound, descriptor, timeline, and sequence builders.
- Paper platform bootstrap, events, scheduler, integrations, command registration, and GUI lifecycle.

## Real server workflows

The docs include compile-checked recipe plugins that show full workflows instead of isolated snippets:

- [Economy plugin](/recipes/economy-plugin/) - commands, typed config, cached data, async persistence, top balances, and admin tools.
- [Player profiles](/recipes/player-profiles/) - join loading, quit saving, profile cache, GUI display, and disable flush.
- [Server menu](/recipes/server-menu/) - real GUI navigation, config-backed buttons, pagination, teleport confirmation, and profile pages.
- [Staff tools](/recipes/staff-tools/) - staff mode, freeze, inspect menus, staff chat, effects, logging, and transient state cleanup.
- [Crate system](/recipes/crate-system/) - config-driven reward tables, key PDC markers, weighted rolls, preview menus, and reward delivery.
- [Kits plugin](/recipes/kits-plugin/) - config-backed kits, cooldown persistence, preview menus, PDC markers, and safe give/drop behavior.

## Migrating an existing plugin

When you are moving an older plugin onto Sculk, start with the [Migration to Sculk 3](/advanced/migration-to-sculk-3/) guide and keep the [Migration Checklist](/advanced/migration-checklist/) open during testing. The migration path is designed around one subsystem at a time: platform lifecycle, text, items, GUI, config, commands, data, events, and optional integrations.
