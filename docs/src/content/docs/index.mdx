Sculk Studio targets Paper 26.1.2+, Java 25, Adventure, and MiniMessage. It is designed for plugin projects that want Kotlin-first APIs without hiding Paper's own model.

<CardGrid>
  <Card title="Commands" icon="list-format">

  </Card>
  <Card title="GUI Menus" icon="laptop">
    Build chest menus with per-player sessions, click routing, pagination, and platform cleanup.
  </Card>
  <Card title="Typed Config" icon="setting">

  </Card>
  <Card title="Data" icon="open-book">

## What is implemented today

- Command DSL and Java command builder.
- GUI menu DSL, sessions, pagination, and platform cleanup.
- MiniMessage and Adventure messaging helpers.
- Typed YAML config loading, comments, validation annotations, and reload callbacks.
- JDBC repositories, storage config, ORM mapping, and cache helpers.
- Registry-backed lookup helpers through `sculk-series`.
- Particle, sound, timeline, and sequence builders.
- Paper platform bootstrap, events, scheduler, command registration, and GUI lifecycle.

## What is planned

- Standalone item builder APIs.
- Paper command API / Brigadier research.
- Focused migration helpers for common XSeries-style aliases.
- More integration tests around platform lifecycle behavior.
