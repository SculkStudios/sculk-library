---
title: Integrations
description: Optional adapters for common server plugins.
---

`sculk-integrations` provides lightweight adapters for common plugins without making them required dependencies.

Enable integrations through the platform:

```kotlin
sculk = SculkPlatform.create(this) {
    integrations()
}
```

## PlaceholderAPI

```kotlin
val papi = sculk.integrations.placeholderApi()
papi.onSuccess { integration ->
    player.sendMini(integration.parse(player, "<green>Hello %player_name%"))
}
```

## Vault Economy

```kotlin
sculk.integrations
    .vaultEconomy()
    .onSuccess { economy ->
        economy.deposit(player, 100.0)
    }
```

## LuckPerms

```kotlin
sculk.integrations
    .luckPerms()
    .onSuccess { luckPerms ->
        val group = luckPerms.primaryGroup(player.uniqueId)
    }
```

Unavailable plugins return `SculkResult.Failure`; they do not crash plugin startup.

## Packet backends

PacketEvents and ProtocolLib are handled by packet modules, not `sculk-integrations`, because they need their own size and lifecycle policy.

- Use `sculk-packets-packetevents` for new packet-heavy plugins.
- Use `sculk-packets-protocollib` when a server already depends on ProtocolLib.
- Install PacketEvents or ProtocolLib on the server; Sculk does not shade them.

See the packet docs for backend setup and thread-safety rules.
