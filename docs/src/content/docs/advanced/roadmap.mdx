Sculk Studio is pre-public, so breaking API cleanup can still happen before the first stable release.

## Implemented Today

- Command DSL and Java command builder.
- GUI menu DSL, sessions, pagination, and platform cleanup.
- MiniMessage and Adventure messaging helpers.
- Typed YAML config loading and validation annotations.
- JDBC repositories, storage config, ORM mapping, and cache helpers.
- Registry helpers through `sculk-series`.
- Particle, sound, timeline, and sequence builders.
- Paper platform bootstrap, events, scheduler, command registration, and GUI lifecycle.
- Version parser for legacy `1.x.y` and new `26.x.y` Minecraft versions.

## Before Public Release

- Complete package/domain migration to `studio.sculk`.
- Harden Paper 26.1.2 support and Java 25 builds.
- Expand command docs with error handling and suggestions.
- Add item builder docs only after the standalone item API exists.
- Add storage examples that show async loading and shutdown saves.
- Add more tests for platform lifecycle behavior.

## Long Term

- Dedicated item module.
- More compatibility mappings.
- Optional Paper command API or Brigadier integration.
- More real-world example plugins.
- Published API docs and Maven metadata.
