Sculk Studio is still pre-public, so API cleanup can happen before the first stable release.

## Implemented today

- Command DSL and Java command builder.
- GUI menu DSL, sessions, pagination, and platform cleanup.
- MiniMessage and Adventure messaging helpers.
- Typed YAML config loading, comments, validation annotations, and reload callbacks.
- JDBC repositories, storage config, ORM mapping, and cache helpers.
- Registry-backed lookup helpers through `sculk-series`.
- Particle, sound, timeline, and sequence builders.
- Paper platform bootstrap, events, scheduler, command registration, and GUI lifecycle.
- Minecraft version parser for legacy `1.x.y` and new `26.x.y` formats.

## Production hardening

- `studio.sculk` package and `sculk.studio` domain migration.
- Paper `26.1.2.build.+` and Java 25 build target.
- Modern Paper registry access.
- Modern item custom model data component usage.
- Explicit Kotlin `@param:` validation annotation examples.
- Docs that distinguish implemented behavior from planned behavior.

## Before public release

- Add more lifecycle tests for GUI registry cleanup.
- Add command dispatch tests for permission and tab-completion edge cases.
- Expand storage examples for async load/save and shutdown flushes.
- Review public API names one more time before declaring stability.
- Publish API docs or link generated Dokka output when configured.

## v1.0 ideas

- Standalone item builder module.
- Focused compatibility alias tables for common migration cases.
- Optional Paper command API / Brigadier integration.
- Dedicated integration-test server module.
- More complete real-world example plugins.
