## Dependency Not Found

Use both JitPack and Paper's Maven repository:

```kotlin
repositories {
    mavenCentral()
    maven("https://jitpack.io")
    maven("https://repo.papermc.io/repository/maven-public/")
}
```

## Paper Or Java Version Errors

Sculk Studio targets Paper 1.21.11+ and Java 21.

```kotlin
compileOnly("io.papermc.paper:paper-api:1.21.11-R0.1-SNAPSHOT")
```

Example `paper-plugin.yml`:

```yaml
name: MyPlugin
version: 1.0.0
main: com.example.myplugin.MyPlugin
api-version: '1.21.11'
```

## Library Not Found At Runtime

Sculk Studio is a library, not a server plugin. Shade it into your plugin jar:

```kotlin
tasks.shadowJar {
    archiveClassifier = ""
    relocate("studio.sculk", "your.plugin.libs.sculk")
}
```

If multiple plugins use Sculk Studio without relocation, class conflicts are possible.

## Config Validation Does Not Run

In Kotlin config data classes, validation annotations should target constructor parameters:

```kotlin
@ConfigFile("settings.yml")
data class Settings(
    @param:Min(1) @param:Max(20) val maxHomes: Int = 5,
    @param:NotEmpty val prefix: String = "<gray>[Plugin]",
)
```

Java records can use the normal annotation form on record components.

## Registry Lookup Returns Null

`SculkSeries` returns `null` when the key is not present in the running server's registry or cannot be normalized.

Check:

- Spelling.
- Whether the key exists in Paper 1.21.11.
- Whether you are using a modern Minecraft key, not an old XSeries alias.
- Whether a fallback should be configured.

```kotlin
val material = SculkSeries.material(config.icon) ?: Material.STONE
```

## GUI Clicks Do Not Fire

Enable the GUI subsystem:

```kotlin
class MyPlugin : SculkPlugin({ gui() })
```

The platform layer owns inventory click/close listeners and session cleanup.

## Async Access Errors

Do not call Paper player, world, inventory, or entity APIs from async work. Use async for database/file/HTTP work, then return to sync:

```kotlin
sculk.scope.launchAsync {
    val profile = repo.find(player.uniqueId)   // suspends off-thread
    sculk.scheduler.runSync(player) {
        player.sendRichMessage("Loaded profile.")
    }
}
```

## Docs Build Or Deploy Fails

The docs site is built from `docs/`.

```bash
bun install
bun run build
```

Cloudflare Workers deploy uses `bunx wrangler deploy`. Confirm these secrets exist in GitHub Actions:

- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`
