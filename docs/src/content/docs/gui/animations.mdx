Sculk 4.0 adds live, coroutine-driven GUI features on top of the existing `gui { }` builder.

## Animated slots

`animate(intervalTicks) { }` cycles a slot through frames while the GUI is open. The animation is
driven by the platform coroutine scope and cancels automatically when the player closes the GUI:

```kotlin
gui("<dark_aqua>Menu") {
    item(22) {
        animate(intervalTicks = 10) {
            frame(Material.RED_WOOL)
            frame(Material.YELLOW_WOOL)
            frame(Material.GREEN_WOOL)
            frame(Material.BLUE_WOOL)
        }
    }
}
```

## Per-click-type handlers

Alongside `onClick { }`, you can register handlers for specific click types — the most specific one
wins:

```kotlin
item(15) {
    material = Material.BARRIER
    name = "<red>Close"
    onLeftClick { close() }
    onRightClick { actionbar("<gray>Right-click does nothing.") }
    onShiftClick { reply("<gray>Shift!") }
}
```

## Interactive input slots

By default every slot is click-locked. Mark a slot `interactive()` to let players freely take from
and place into it — useful for deposit boxes and input menus. Drags are allowed only when every
affected top slot is interactive.

```kotlin
gui("<gold>Deposit Box") {
    type = InventoryType.HOPPER
    item(0) { material = Material.GRAY_STAINED_GLASS_PANE }
    item(4) { material = Material.GRAY_STAINED_GLASS_PANE }
    for (slot in 1..3) item(slot) { interactive() }
}
```

## Non-chest containers

Set `type` to any container `InventoryType` (hopper, dispenser, …) — the size adjusts automatically:

```kotlin
gui("<gold>Tools") {
    type = InventoryType.DISPENSER  // 9-slot 3x3 grid
    // ...
}
```

## Live updates

`session.renderSlot(slot, stack)` updates a single slot in place without reopening — the primitive
behind animations and any custom live behaviour.
