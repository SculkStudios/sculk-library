---
title: Pagination
description: Distribute items across multiple GUI pages automatically.
---

When you have more entries than slots, use `pagination { }` to distribute them across pages.

## Basic pagination

```kotlin
val shopMenu = gui("<green><bold>Shop") {
    size = 54

    pagination {
        slots += (0 until 45).toList()  // top 5 rows hold items
    }

    item(45) {
        material = Material.ARROW
        name = "<gray>← Previous"
        onClick { previousPage() }
    }

    item(53) {
        material = Material.ARROW
        name = "<gray>Next →"
        onClick { nextPage() }
    }
}
```

## Supplying entries

```kotlin
player {
    val session = shopMenu.openFor(player!!)
    session.setEntries(getShopItems())
}
```

Entries are distributed across the defined slots automatically. Navigating with `previousPage()` / `nextPage()` re-renders the current page in-place with no inventory flicker. Use `session.setPage(n)` to jump directly to a specific page (0-indexed, clamped to the valid range):

```kotlin
onClick {
    session.setPage(0)  // jump to first page
}
```

## Dynamic refresh

Update a single slot without reopening the inventory:

```kotlin
session.refresh(slot)
```

Re-render all slots at once:

```kotlin
session.refreshAll()
```

## Use cases

- Shop browsers, auction houses, player list menus
- Any list longer than a single page of slots
