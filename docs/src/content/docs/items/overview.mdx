`sculk-items` is the standalone item API for Sculk Studio 2.0. It is used directly in plugins and by GUI builders for shared item metadata behavior.

```kotlin
val sword = item(Material.DIAMOND_SWORD) {
    name("<aqua>Starter Sword")
    lore("<gray>A clean starter weapon.", "", "<yellow>Right-click to inspect.")
    amount(1)
    enchant(Enchantment.SHARPNESS, 5)
    glint()
    unbreakable()
    customModelData(1001)
    pdc("starter_item", true)
}
```

String material keys are supported for config-driven workflows:

```kotlin
val icon = item("nether_star") {
    name("<aqua>Profile")
    lore("<gray>Click to view your profile.")
}
```

## GUI Integration

GUI items can use the same builder through `stack { }`:

```kotlin
item(13) {
    stack {
        material("nether_star")
        name("<aqua>Profile")
        lore("<gray>Click to view your profile.")
        customModelData(1001)
    }

    onClick {
        reply("<green>Opening profile...")
    }
}
```

```kotlin
val head = skull {
    owner(player.uniqueId)
    name("<aqua>${player.name}")
    lore("<gray>Built with Sculk skull().")
}
```

## Safe Give Or Drop

Use `giveOrDrop` when rewards must never vanish because a player's inventory is full:

```kotlin
val reward = item("diamond") {
    name("<aqua>Vote Diamond")
}

val result = player.giveOrDrop(reward)

if (result.dropped.isNotEmpty()) {
    player.sendMessage("Some rewards were dropped at your feet.")
}
```

Call this from the server thread, or from the player's region thread on Folia-like servers. If the item is part of a database-backed reward flow, load and decide the reward asynchronously, then hand back to sync before touching the inventory.

## Java

```java
ItemStack sword = JavaItemBuilder.of(Material.DIAMOND_SWORD)
    .name("<aqua>Starter Sword")
    .lore("<gray>A clean starter weapon.")
    .enchant(Enchantment.SHARPNESS, 5)
    .glint()
    .customModelData(1001)
    .pdc("starter_item", true)
    .build();
```
