---
title: Registries Reference
description: Available SculkSeries lookup methods and expected behavior.
---

Each `SculkSeries` method accepts a string key and returns either the resolved Paper value or `null`.

## Registry reference

| Method | Returns | Notes |
| --- | --- | --- |
| `SculkSeries.material(key)` | `Material?` | Uses Paper/Bukkit material matching. |
| `SculkSeries.sound(key)` | `Sound?` | Uses Paper sound event registry. |
| `SculkSeries.particle(key)` | `Particle?` | Uses Paper particle registry. |
| `SculkSeries.enchantment(key)` | `Enchantment?` | Uses Paper enchantment registry. |
| `SculkSeries.entityType(key)` | `EntityType?` | Uses Paper entity type registry. |
| `SculkSeries.potionEffect(key)` | `PotionEffectType?` | Uses Paper mob effect registry. |
| `SculkSeries.biome(key)` | `Biome?` | Uses Paper biome registry. |
| `SculkSeries.gameMode(key)` | `GameMode?` | Matches enum names case-insensitively. |
| `SculkSeries.difficulty(key)` | `Difficulty?` | Matches enum names case-insensitively. |

## Key format

These forms are accepted for registry-backed values:

```text
sharpness
minecraft:sharpness
entity.player.levelup
entity_player_levelup
ENTITY_PLAYER_LEVELUP
```

Input is trimmed, lowercased, and normalized through a small set of namespace and dot/underscore candidates. This does not guarantee old-version alias resolution.

## Materials

```kotlin
val icon = SculkSeries.material(config.iconMaterial) ?: Material.STONE
```

## Sounds

```kotlin
val sound = SculkSeries.sound("entity.player.levelup")
if (sound != null) {
    player.playSound(player.location, sound, 1.0f, 1.0f)
}
```

## Enchantments

```kotlin
val sharpness = SculkSeries.enchantment("sharpness")
if (sharpness != null) {
    sword.addEnchantment(sharpness, 5)
}
```

## Custom registries

Register your own lookup when the built-in list does not cover a type:

```kotlin
SculkSeries.register<MyType> { key ->
    MyType.entries.firstOrNull { it.name.equals(key, ignoreCase = true) }
}

val value = SculkSeries.lookup<MyType>("my_key")
```

## Performance

Lookups are cached by `SculkRegistry`. The first request for a key performs normalization and registry lookup. Later requests for the same key return from the cache.

Pre-warm important config-driven keys in `onEnable` so missing-key warnings can be logged during startup rather than gameplay.
