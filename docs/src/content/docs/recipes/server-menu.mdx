## What This Builds

A practical `/menu` workflow:

- Main menu with config-backed buttons.
- Warps menu with real teleport actions.
- Profile card menu.
- Online players page.
- Confirmation menu for a local setting action.

## Modules Used

```kotlin
implementation("com.github.SculkStudios.sculk-library:sculk-platform:v2.0.0")
implementation("com.github.SculkStudios.sculk-library:sculk-items:v2.0.0")
```

## Commands

| Command | Purpose |
|---|---|
| `/menu` | Opens the main server menu. |

## Permissions

This example has no permission requirement. Add one to the command root for staff-only or donor-only menus.

## Config

```kotlin
@ConfigFile("menus.yml")
data class MenuSettings(
    val title: String = "<dark_gray>Server Menu",
    val buttons: MenuButtons = MenuButtons(),
    val warps: List<WarpDefinition> = listOf(...),
)
```

Buttons use `ItemDescriptor`, so server owners can edit material, name, lore, glint, model data, and PDC in config.

## Data Model

This recipe is intentionally config-driven. It does not persist player data.

## Lifecycle

```kotlin
sculk = SculkPlatform.create(this) {
    config()
    gui()
}

settings = sculk.config.load()
menus = ServerMenus { settings }
```

The `gui()` subsystem is required. It routes clicks, drags, close events, and player quit cleanup.

## Implementation Walkthrough

The main menu uses real click actions:

```kotlin
descriptorItem(10, settings().buttons.warps) {
    open(warps())
}

descriptorItem(12, settings().buttons.profile) {
    open(profile(player))
}
```

The warps menu validates that the world is loaded before teleporting.

## Threading Rules

- GUI clicks run on the server thread.
- Do not perform database or HTTP work directly in click handlers.
- Teleport and inventory work must stay sync.
- If a menu needs database data, load it async first and open/update the GUI after a sync handoff.

## Failure Behavior

- Missing worlds produce an error message instead of throwing.
- Empty online player lists render a fallback item.
- Confirm actions close the session after feedback.

## Common Mistakes

- Forgetting `gui()` in `SculkPlatform.create`.
- Using placeholder buttons that do not perform real actions.
- Loading data directly inside click handlers.
- Reusing one mutable menu state object for every player.
