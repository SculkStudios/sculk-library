This recipe mirrors `examples/economy-plugin`.

Modules:

```kotlin
implementation("com.github.SculkStudios.sculk-library:sculk-platform:v2.0.0")
implementation("com.github.SculkStudios.sculk-library:sculk-items:v2.0.0")
```

## Config

```kotlin
@ConfigFile("economy.yml")
data class EconomySettings(
    val startingCoins: Long = 100,
    val balanceMessage: String = "<gold>You have <yellow><coins></yellow> coins.",
)
```

## Command

```kotlin
sculk.commands.register(
    command("coins") {
        description = "View and manage coins."
        aliases = listOf("money", "balance")

        player {
            val balance = balances.computeIfAbsent(player!!.uniqueId) { settings.startingCoins }
            reply(settings.balanceMessage.replace("<coins>", balance.toString()))
        }

        sub("give") {
            permission = "economy.admin"
            player("target")
            long("amount", min = 1)
            executes {
                val target = argument<Player>("target")
                val amount = argument<Long>("amount")
                balances.merge(target.uniqueId, amount, Long::plus)
                reply("<green>Gave <yellow>$amount</yellow> coins to <aqua>${target.name}</aqua>.")
            }
        }
    },
)
```

Common mistakes:

- Keep blocking database-backed balance repositories off the main thread.
- Put admin mutations behind permissions.
- Keep messages in config once your plugin needs localization.
