Sculk Studio is distributed through [JitPack](https://jitpack.io). **One dependency is all you need:**
`sculk-platform` transitively re-exports the entire DSL — commands, GUI, items, config, data, text,
tasks, events, and scheduling. You do **not** add `sculk-items`, `sculk-config`, etc. separately.

Worried about jar size? Don't be — `shadowJar { minimize() }` strips every class you don't use, so a
plugin that only touches commands and GUIs ships a tiny jar even though `sculk-platform` pulls the
full graph. Heavy optional pieces (MySQL driver, Redis, packet backends) are **not bundled** unless
you opt in (see below).

## Gradle (Kotlin DSL)

Add JitPack to your repository list and declare the dependency:

```kotlin
// settings.gradle.kts
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        mavenCentral()
        maven("https://jitpack.io")
        maven("https://repo.papermc.io/repository/maven-public/")
    }
}
```

```kotlin
// build.gradle.kts
plugins {
    kotlin("jvm") version "2.3.21"
    id("com.gradleup.shadow") version "9.4.1"
}

dependencies {
    compileOnly("io.papermc.paper:paper-api:1.21.11-R0.1-SNAPSHOT")

    // The only dependency you need — re-exports the whole DSL.
    implementation("com.github.SculkStudios.sculk-library:sculk-platform:4.0.0")
}

tasks.shadowJar {
    archiveClassifier = ""
    minimize()  // strip unused classes — keeps the shaded jar small
    relocate("studio.sculk", "your.plugin.name.libs.sculk")
}
```

## Gradle (Groovy DSL)

```groovy
// settings.gradle
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        mavenCentral()
        maven { url 'https://jitpack.io' }
        maven { url 'https://repo.papermc.io/repository/maven-public/' }
    }
}
```

```groovy
// build.gradle
plugins {
    id 'com.gradleup.shadow' version '9.4.1'
}

dependencies {
    compileOnly 'io.papermc.paper:paper-api:1.21.11-R0.1-SNAPSHOT'
    implementation 'com.github.SculkStudios.sculk-library:sculk-platform:4.0.0'
}

shadowJar {
    archiveClassifier = ''
    minimize()
    relocate 'studio.sculk', 'your.plugin.name.libs.sculk'
}
```

## Shading is required

Sculk Studio must be shaded into your plugin jar. It is not a server plugin; it is a library. The `shadowJar` configuration above handles this. Always relocate the `studio.sculk` package to avoid conflicts if multiple plugins on the same server use Sculk Studio.

Replace `your.plugin.name` in the relocation path with your own package name (e.g., `com.example.myplugin.libs.sculk`).

## plugin.yml / paper-plugin.yml

No special entries are needed in your plugin descriptor for normal Sculk modules. They shade into your jar and have no external server dependencies.

```yaml
# paper-plugin.yml
name: MyPlugin
version: 1.0.0
main: com.example.myplugin.MyPlugin
api-version: '1.21.11'
```

## Jar size & `minimize()`

`sculk-platform` pulls a broad dependency graph, but `shadowJar { minimize() }` removes every class
your plugin never references, so your output jar only contains what you actually use.

**Caveat:** `minimize()` strips classes that are only loaded by reflection or by name — which
includes the bundled **SQLite driver**. If your plugin uses the data layer (`data()`), keep the
driver:

```kotlin
tasks.shadowJar {
    minimize {
        exclude(dependency("org.xerial:sqlite-jdbc:.*")) // loaded by class name at runtime
    }
}
```

If you don't use `data()` at all, plain `minimize()` is fine — the whole data layer is stripped.

## Opt-in heavy dependencies

These are **not bundled** to keep jars small. Add them only when you use the feature:

```kotlin
dependencies {
    // MySQL/MariaDB storage (default backend is SQLite, already bundled)
    runtimeOnly("org.mariadb.jdbc:mariadb-java-client:3.5.8")

    // Redis distributed cache (data.distributedCache)
    implementation("io.lettuce:lettuce-core:6.7.1.RELEASE")
}
```

## Optional packet modules

Packet support is intentionally separate so normal plugins stay small. The packet **API** already
comes with `sculk-platform`; to enable packets, add **one backend adapter** (it pulls the API in
transitively). Sculk does not bundle PacketEvents or ProtocolLib themselves.

PacketEvents path:

```kotlin
dependencies {
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-packetevents:4.0.0")
}
```

ProtocolLib path:

```kotlin
dependencies {
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-protocollib:4.0.0")
}
```

Install PacketEvents or ProtocolLib on the server and add the matching soft dependency if your plugin requires it:

```yaml
softdepend:
  - packetevents
  - ProtocolLib
```

## Verifying the install

After adding the dependency, create a minimal plugin class and confirm it compiles:

```kotlin
import studio.sculk.platform.SculkPlugin

class MyPlugin : SculkPlugin() {
    override fun setup() {
        logger.info("Sculk Studio loaded.")
    }
}
```

If it compiles and the server logs `Sculk Studio loaded.` on startup, you're ready to go. Head to [Your First Plugin](/getting-started/first-plugin) for the full walkthrough.
