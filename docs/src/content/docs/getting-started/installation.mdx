---
title: Installation
description: Add Sculk Studio to your Minecraft Paper plugin project via JitPack.
---

Sculk Studio is distributed through [JitPack](https://jitpack.io). Install only the modules your plugin uses. `sculk-platform` wires lifecycle, commands, scheduler, GUI routing, config, data, integrations, and optional packet services, but packet backend adapters stay separate.

## Gradle (Kotlin DSL)

Add JitPack to your repository list and declare the dependency:

```kotlin
// settings.gradle.kts
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        mavenCentral()
        maven("https://jitpack.io")
        maven("https://repo.papermc.io/repository/maven-public/")
    }
}
```

```kotlin
// build.gradle.kts
plugins {
    kotlin("jvm") version "2.3.21"
    id("com.gradleup.shadow") version "9.4.1"
}

dependencies {
    compileOnly("io.papermc.paper:paper-api:26.1.2.build.+")

    implementation("com.github.SculkStudios.sculk-library:sculk-platform:v2.0.0")
    implementation("com.github.SculkStudios.sculk-library:sculk-items:v2.0.0")
}

tasks.shadowJar {
    archiveClassifier = ""
    relocate("studio.sculk", "your.plugin.name.libs.sculk")
}
```

## Gradle (Groovy DSL)

```groovy
// settings.gradle
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        mavenCentral()
        maven { url 'https://jitpack.io' }
        maven { url 'https://repo.papermc.io/repository/maven-public/' }
    }
}
```

```groovy
// build.gradle
plugins {
    id 'com.gradleup.shadow' version '9.4.1'
}

dependencies {
    compileOnly 'io.papermc.paper:paper-api:26.1.2.build.+'
    implementation 'com.github.SculkStudios.sculk-library:sculk-platform:v2.0.0'
    implementation 'com.github.SculkStudios.sculk-library:sculk-items:v2.0.0'
}

shadowJar {
    archiveClassifier = ''
    relocate 'studio.sculk', 'your.plugin.name.libs.sculk'
}
```

## Shading is required

Sculk Studio must be shaded into your plugin jar. It is not a server plugin; it is a library. The `shadowJar` configuration above handles this. Always relocate the `studio.sculk` package to avoid conflicts if multiple plugins on the same server use Sculk Studio.

Replace `your.plugin.name` in the relocation path with your own package name (e.g., `com.example.myplugin.libs.sculk`).

## plugin.yml / paper-plugin.yml

No special entries are needed in your plugin descriptor for normal Sculk modules. They shade into your jar and have no external server dependencies.

```yaml
# paper-plugin.yml
name: MyPlugin
version: 1.0.0
main: com.example.myplugin.MyPlugin
api-version: '26.1.2'
```

## Optional packet modules

Packet support is intentionally separate so normal plugins stay small. Sculk does not bundle PacketEvents or ProtocolLib.

PacketEvents path:

```kotlin
dependencies {
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-api:v2.0.0")
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-packetevents:v2.0.0")
}
```

ProtocolLib path:

```kotlin
dependencies {
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-api:v2.0.0")
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-protocollib:v2.0.0")
}
```

Install PacketEvents or ProtocolLib on the server and add the matching soft dependency if your plugin requires it:

```yaml
softdepend:
  - packetevents
  - ProtocolLib
```

## Verifying the install

After adding the dependency, create a minimal plugin class and confirm it compiles:

```kotlin
import studio.sculk.platform.SculkPlatform
import org.bukkit.plugin.java.JavaPlugin

class MyPlugin : JavaPlugin() {
    lateinit var sculk: SculkPlatform

    override fun onEnable() {
        sculk = SculkPlatform.create(this) {}
        logger.info("Sculk Studio loaded.")
    }

    override fun onDisable(): Unit = sculk.close()
}
```

If it compiles and the server logs `Sculk Studio loaded.` on startup, you're ready to go. Head to [Your First Plugin](/getting-started/first-plugin) for the full walkthrough.
