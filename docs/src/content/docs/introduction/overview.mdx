---
title: What is Sculk Studio?
description: Kotlin-first Paper utilities for modern plugin development.
---

Sculk Studio is a Kotlin-first library for Paper plugins. It provides focused APIs for command trees, GUI menus, typed configuration, data access, Adventure text, scheduling, events, effects, and Paper lifecycle wiring.

The library keeps Paper visible. It does not try to replace Paper, hide thread-safety rules, or emulate every legacy compatibility library. Instead, it gives you smaller APIs around the work most Paper plugins repeat.

## Example

```kotlin
@ConfigFile("settings.yml")
data class Settings(
    @param:Min(1) val maxHomes: Int = 5,
)

val homeMenu = gui("<dark_aqua>Your Homes") {
    size = 27

    item(13) {
        material = Material.COMPASS
        name = "<aqua>Set Home"
        onClick {
            reply("<green>Home set.")
            close()
        }
    }
}

fun homeCommand(sculk: SculkPlatform) = command("home") {
    permission = "homes.use"

    sub("set") {
        string("name")
        player {
            val name = argument<String>("name")
            val settings = sculk.config.load<Settings>()
            reply("<green>Home <white>$name</white> set. Max homes: ${settings.maxHomes}")
        }
    }

    sub("menu") {
        player {
            homeMenu.openFor(player!!)
        }
    }
}
```

## Design goals

**Kotlin-first.** The main API uses Kotlin DSLs where they improve readability.

**Java-usable.** Core systems expose Java builders or Java-friendly entry points where practical.

**Paper-first.** Sculk Studio targets modern Paper 26.1.2+ and Java 25.

**Adventure-first.** Message helpers use Adventure components and MiniMessage, not legacy color-code-first APIs.

**Explicit thread rules.** Repository calls are synchronous and blocking. Run them off the main thread, then return to the correct Paper thread before touching players, worlds, or inventories.

**Small modules.** Each module has a focused job and can be reasoned about independently.

## Modules

Sculk is split into focused modules — `sculk-commands`, `sculk-gui`, `sculk-config`, `sculk-items`,
`sculk-data`, `sculk-text`, and more — all built on a slim `sculk-common` base. Depend on the single
`sculk-platform` for everything, or pick individual modules à la carte.

See [Modules & Architecture](/introduction/modules/) for the full section ↔ module ↔ package map and
the dependency graph.

## What Sculk Studio is not

- It is not a replacement for Paper's API.
- It is not a full XSeries clone.
- It is not a standalone server plugin. Shade and relocate it into your plugin jar.
