title: PacketEvents
description: Recommended backend for new packet-heavy Sculk plugins.

PacketEvents is the preferred packet backend for new packet-heavy Sculk plugins. It is designed around packet listeners, priorities, wrapper APIs, and explicit packet mutation behavior.

The Sculk PacketEvents adapter is kept separate from core. It supports backend detection, low-level listener registration, listener cleanup, packet cancellation, changed-packet marking, and sending `PacketEventsPacket` wrappers.

## Dependency

```kotlin
dependencies {
    implementation("com.github.SculkStudios.sculk-library:sculk-platform:4.0.0")
    // The backend adapter pulls the packet API in transitively.
    implementation("com.github.SculkStudios.sculk-library:sculk-packets-packetevents:4.0.0")
}
```

Install PacketEvents on the server. Do not shade PacketEvents into your plugin.

```yaml
softdepend:
  - packetevents
```

## Backend selection

```kotlin
class MyPlugin : SculkPlugin({
    packets {
        backend = PacketBackendMode.PacketEvents
        required = true
    }
})
```

Use `required = true` only when the plugin cannot run without packet features.

## Low-level listener

```kotlin
val packets = sculk.packets

packets.listen(PacketDirection.Serverbound, PacketKey.of("use_item"), PacketPriority.Normal) {
    val source = player ?: return@listen
    runSync {
        source.sendMessage("Use item packet received.")
    }
}
```

If you modify a PacketEvents wrapper inside a listener, call `markChanged()` so PacketEvents re-encodes the packet.

## PacketEvents rules

- Packet callbacks may not be on the main thread.
- Use `context.runSync { ... }` for Paper API access.
- When a wrapper is modified, call `context.markChanged()` so the adapter maps that to PacketEvents re-encoding behavior.
- Avoid heavy allocations in hot packet paths.
- Prefer packet type filters before decoding packet data.

## When to use it

PacketEvents is the right default for:

- Packet-heavy cosmetics.
- Client block previews.
- Per-player visibility logic.
- Debugging or inspecting high-frequency packet flows.

Use Paper APIs instead when they already provide the behavior with normal server state.
