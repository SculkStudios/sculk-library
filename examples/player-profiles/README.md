# Player Profiles Recipe

Compile-checked example for loading, caching, displaying, saving, and flushing player profiles.

## Shows

- UUID-first profile data with `sculk-data`.
- Join load and quit save lifecycle.
- Async repository calls with sync player handoff.
- A profile GUI built from loaded data.
- Disable-time flush for loaded profiles.

## Commands

| Command | Purpose |
|---|---|
| `/profile` | Open your profile menu. |
| `/profile <player>` | Open another online player's profile menu. |
| `/profile reload` | Demonstrates command structure for reload workflows. |
